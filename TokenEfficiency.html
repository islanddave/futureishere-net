🪙 The basics
🪙
What Is a Token?
The unit that drives everything — cost, speed, and quality
+
TL;DRA token is roughly ¾ of a word. Every token in your prompt and every token in Claude's response counts toward your usage. Input and output tokens are both billed — output is typically billed higher.
Quick estimates
≈ 1.3tokens per word
15–25tokens per sentence
500–600tokens per page
750words per 1,000 tokens
What counts toward your usage
1
Your prompt (input tokens) Every word in your message — including pasted documents, conversation history, and system instructions — counts as input.
2
Claude's response (output tokens) Output tokens are typically billed at 3–5× the input rate on most API plans. Controlling response length is high-leverage.
3
Context window on every turn In a multi-turn conversation, the entire conversation history is resent to Claude on each turn. A long thread costs more with every message you add.
💡
The biggest surprise: context is cumulative. A 10-message thread doesn't cost 10× one message — it costs the sum of all messages × 10 (once per turn). Long conversations compound fast.

Write Leaner Prompts
Cut filler · use imperatives · specify format upfront
+
TL;DREvery unnecessary word is a wasted token. Start with action verbs. Remove filler phrases. Tell Claude the format before you give the task — it prevents hedging and round-trips.
Cut these phrases entirely
✗ With filler✓ Direct

I would like you to please make a worksheet for my students.Create a gap-fill: 10 sentences, present simple, A2 level. Include answer key.
Could you possibly check if this activity is level-appropriate?Is this activity A2 or B1? Answer in 2 sentences.
Then add a format spec

"One paragraph, no preamble."
"Return only valid JSON — no explanation."

Manage Context Strategically
Trim history · summarise docs · use references not repetition
+
TL;DRContext — everything the model sees before generating — is the largest driver of token usage. It's resent on every turn. Trim ruthlessly: summaries instead of threads, excerpts instead of full documents, references instead of re-pastes.
Trim conversation history
✗ Three follow-ups in old thread✓ One opener in fresh chat
User: "also can you add a speaking activity" "Lesson: 45 min, B1, daily routines.
Need: reading text + 5 Qs + speaking task + answer key.
No preamble, ready to print."
User: "and make it work for mixed ability beginners"
User: "oh and include the answer key too"
Use references, not repetition
🔗
Shorthand that works: "the lesson above" · "that worksheet" · "the previous summary" · "the vocab list from step 2"

Each re-paste can add 100–500 tokens per turn.

⚙️ Tip 03
⚙️
Write Efficient System Prompts
DRY · compress with examples · cache-first structure
+
⏭️
API users only This section is for developers building with the Claude API. If you're using claude.ai in your browser, you don't have a system prompt — safe to skip.
TL;DRSystem prompts run on every API call — a bloated one multiplies your cost by total request volume. Keep rules numbered and short. Replace verbose prose with a single example. Put static content first so prompt caching can eliminate up to 90% of those tokens.
DRY: Don't Repeat Yourself
✗ Verbose prose✓ Compressed
Always respond in a way that is helpful and appropriate for ESL teachers working with adult learners.Audience: ESL teachers, adult learners, B1–B2.
Make sure your language is not too complex and avoid using idioms that non-native speakers might not know.Example: "Here's a warmup activity:"
Be encouraging and supportive at all times and keep things clear.Avoid: native-speaker idioms. Max: 3 sentences per explanation.
Cache-friendly prompt structure (API)
📦
Optimal order:
[1] Static system instructions ← cached
[2] Static reference documents ← cached
[3] Dynamic user message ← always billed

Static content placed after dynamic content cannot be cached.

📏 Tip 04
📏
Control Response Length
Set limits · batch questions · request structured data
+
TL;DROutput tokens are the most expensive. An open-ended request invites a long response. Set explicit length limits, batch related questions into single prompts, and request structured data instead of written explanations when you just need the information.
Set explicit length limits
📐
Add limits inline:
"Answer in one sentence." · "Max 200 words."
"No preamble." · "No explanation — code only."

In API calls: set max_tokens to hard-cap output. Even a loose cap (e.g. 500) prevents runaway responses.
Batch related questions
✗ Three separate calls✓ One batched prompt
What is present perfect? "Present perfect vs past simple — 2 sentences each.
Then 3 classroom examples.
No preamble."
When do I use it instead of past simple?
Can you give me some examples to use in class?
